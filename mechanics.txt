Comprehensive Mechanics Sheet

Launch flow

Beginner entrypoint is runme.py.

Runtime orchestration lives in LIMain.py.

Expected startup path: runme intro -> LIMain.ethical_boot_sequence() -> consentform -> LIMain.session() -> shutdown save in LIMain.main().

Session lifecycle

Boot sequence initializes SQLite through database.init_db() and constructs the narrator as Me(persona="basic").

The boot sequence asks for a username, seeds deterministic personality variance with theatrics.seed_from_username(), and attempts to reload prior persona, closeness, and slip state through database.load_session().

Slip intensity decays by elapsed time away before the next run begins.

Consent is collected through ConsentKey.display() and ConsentKey.get_consent() in consentform.py.

If consent is denied, LIMain closes the DB connection and returns before any collection stage begins.

If consent is granted, LIMain.session() runs the collection stages in this order: file enumeration, system profiling, service/process detection, autosave flush, end-of-session state save.

Data collection mechanics

File enumeration is attempted first through enumeration.FileCrawler.collect_and_log().

System profile snapshot is assembled in LIMain.system_profiler().

Service/process detection runs in services.prog().

Collected stage outputs are narrated through theatrics.equip(...), which adds pieces to the persona and can buffer flat fields into autosave.

Narration and persona engine

Persona state object is Me in theatrics.py.

Quip lookup and fallback logic combine catalog-backed lookup with DB-backed rows in theatrics.py and quips.py.

Instability and slip behavior are handled in theatrics.py.

Persona escalation is driven by closeness and slip thresholds in theatrics.py.

Persistence mechanics

SQLite schema, evidence directory handling, session load, and session save live in database.py.

Session rows are keyed by canonicalized username, with the stored row id derived from session_id plus the normalized username.

Buffered write behavior and retry flow are in autosave.py.

End-of-session state persistence happens in LIMain.main() inside the finally block so normal exits still attempt a save.

Stage contract for future modules

Return a flat key-value mapping when a stage wants to feed narration/autosave directly.

Log through existing helpers where a stage owns its own DB writes.

Pass the autosave object through stage calls when incremental persistence is desired.

Keep new stage outputs compatible with theatrics.equip(...), which expects a flat mapping.

Current boot-sequence notes

The opening hello state is still narrative state, not a direct log row; theatrics.equip(...) narrates and autosaves when an autosave object is present.

The consent-request event is intentionally logged once through a single explicit log(...) call in LIMain.ethical_boot_sequence().

Current operational constraints

Main path is interactive and consent-gated.

Validation is currently targeted/manual rather than CI-driven.

Session internals are intentionally debug/testing-oriented for now.
